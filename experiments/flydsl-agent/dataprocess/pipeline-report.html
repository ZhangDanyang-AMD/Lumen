SFT v2 Data Enhancement: 19% → 59% Kernel Code

SFT v1 experiment revealed that 82% of training data contained no FlyDSL kernel code, causing the model to learn document QA instead of kernel generation. v2 adds 108 new kernel pairs from CPT data and applies weighted resampling to triple the kernel code ratio.

Kernel Code Rate: v1 vs v2 3x improvement
Enhancement Details

Step 1: Extract 54 FlyDSL kernel files from CPT data → 108 new SFT pairs (2 styles each)

Step 2: Weighted resampling of 2,916 combined samples:

  • augmentation_tile/pipeline: 5.0x (75% kernel rate)
  • kernel_code_synthesis: high (100% kernel, new data)
  • documentation_qa: 0.3x (8% kernel, downweight)
  • refusal_boundary: 0.1x (0% kernel, minimal)
  • Kernel-containing samples: additional 2.0x bonus

Result: 2,916 samples, 59% kernel code (was 2,808 / 19%)
