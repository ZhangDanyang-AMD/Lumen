Lumen: AMD-Native
Quantized Training for LLMs

A lightweight framework that manages the quantized training lifecycle — forward, backward, optimizer, and communication — with one-line non-invasive patching on AMD Instinct hardware. Now with VERL FSDP2 & Megatron RL, SGLang and vLLM rollout on ROCm.

Key Features

Non-invasive FP8 quantized training with high-performance AMD-native kernels

⚡

Non-invasive Patching

quant.enable(model) patches any model for FP8 training without changing model code or checkpoint format. One line, zero model modifications.

⚙

Unified QuantConfig

A single configuration object governs format (E4M3, E5M2, MXFP8, Hybrid), scaling strategy (Dynamic, Delayed, Blockwise), and all precision knobs.

▲

Up to 2x Peak FLOPS

FP8 on MI300X achieves ~2,615 TFLOPS vs ~1,307 TFLOPS for BF16. End-to-end training speedup of 1.4-1.7x depending on workload.

♺

AITER Kernels

High-performance fused operators with ASM -> CK -> Triton fallback chain. Covers attention, GEMM, normalization, RoPE, MoE, MLP, cross-entropy.

☷

MORI Communication

Native RDMA + GPU communication with FP8 all-gather (~2x bandwidth saving) and compute-communication overlap (AG/fwd, RS/bwd).

⚗

Backend-Agnostic

Same quantization semantics across FSDP, FSDP2, and Megatron-LM backends. Supports SFT, pre-training, and RL workloads (GRPO, PPO, DAPO) with SGLang or vLLM rollout.

Architecture

Lumen owns the quantized training lifecycle and delegates everything else to the training backend

Quick Start

Enable quantized training with a single line of code

Install
# User install
pip install lumen[all]

# Developer install
git clone git@github.com:ZhangDanyang-AMD/Lumen.git
cd Lumen
pip install -e ".[dev]"
Non-invasive Patching
import lumen.quantize as quant
from lumen.quantize import QuantConfig, QuantFormat, ScalingType

# One line — patches model for FP8 training
quant.enable(model, format="fp8_e4m3", scaling="delayed")

# Training loop unchanged
output = model(input_ids)
loss.backward()
optimizer.step()
Full Config
config = QuantConfig(
    format=QuantFormat.FP8_E4M3,
    scaling=ScalingType.DELAYED,
    amax_algo=AmaxAlgo.MAX,
    history_len=16,
    quantize_activation=True,
    quantize_grad="fp8",
)
quant.enable(model, config=config)
RL Training (VERL + FSDP2 + SGLang/vLLM)
# VERL + FSDP2 + SGLang with FP8 Param Manager
# -25% peak VRAM vs BF16, 1.3x faster vs Megatron
FP8_PARAM_MANAGER=1 \
NUM_GPUS=4 ROLLOUT_TP=2 \
bash examples/rl/verl/run_grpo_fsdp2.sh

# Also works with vLLM rollout on ROCm
bash examples/rl/verl/run_grpo_fsdp2_vllm.sh
